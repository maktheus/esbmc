Pipeline de Verificacao Formal — DDPG Cart-Pole

Modelo (Rede Neural) vs Controlador (Malha Fechada) — onde o ESBMC atua

Dominio 1: MODELO (Rede Neural)

Verifica propriedades da rede isolada, sem considerar a planta.
Neuronios mortos: ReLU(pre) == 0 para todo input? → indica neuronio inutil
Saturacao: ReLU(pre) > 0 para todo input? → neuronio nunca desativa
Bounds de saida: |F| <= 10N sempre? → sanidade do Q8.8

Tanh: Para Props de rede, tanh nao entra (so ReLU nas hidden layers). Para Prop C, usa-se monotonicidade: |tanh(z)| <= 1 → |F| <= 10.

Dominio 2: CONTROLADOR (Malha Fechada)

Verifica o sistema completo: rede + dinamica da planta em loop.
Prop A (direcao): theta > 5.7° ∧ theta_dot >= 0 → F > 0?
  Usa monotonicidade: z > 0 ⟺ tanh(z) > 0 ⟺ F > 0
Prop B (seguranca): s0 ∈ S_safe → |theta_new| < 12° apos 1 step?
  Precisa do valor de F → aprox. linear de tanh em Q8.8
  Dinamica linearizada: sin(θ) ≈ θ, cos(θ) ≈ 1

Contraexemplos: estado onde a propriedade falha → injetavel na simulacao.
