\chapter{Fundamentação Teórica}
\label{cap:fundamentacao}

Este capítulo apresenta os conceitos fundamentais que sustentam a verificação formal de software, com foco em Verificação de Modelos Limitada (\textit{Bounded Model Checking} - BMC) e na arquitetura do verificador ESBMC.

\section{Verificação Formal e Model Checking}

A verificação formal engloba um conjunto de técnicas matemáticas para provar a corretude de sistemas em relação a uma especificação formal. Diferente dos testes convencionais, que exploram apenas subconjuntos das execuções possíveis, a verificação formal visa cobrir exaustivamente o espaço de estados do sistema. 

O \textit{Model Checking} é uma técnica automática que verifica se um modelo de um sistema satisfaz uma determinada propriedade lógica (geralmente expressa em lógica temporal). Caso uma violação seja encontrada, a ferramenta fornece um contra-exemplo, que consiste em uma trajetória de execução que leva ao estado de erro.

\section{Bounded Model Checking (BMC)}

O \textit{Bounded Model Checking} é uma técnica baseada em satisfatibilidade lógica (SAT/SMT) onde o sistema é analisado até um limite fixo de passos (\textit{bound} $k$). Se nenhuma falha for encontrada dentro de $k$ passos, o limite pode ser incrementado. O processo envolve:
\begin{enumerate}
    \item Desenrolamento dos laços e chamadas de função até o limite $k$;
    \item Conversão do código e das propriedades em uma fórmula lógica;
    \item Verificação da fórmula por um \textit{solver} SMT.
\end{enumerate}

\subsection{Satisfatibilidade Logica (SAT e SMT)}

O Bounded Model Checking se apoia fortemente em resolvedores (\textit{solvers}) SAT e SMT para determinar a validade das propriedades verificadas.

O problema de \textbf{Satisfatibilidade Booleana (SAT)} busca encontrar uma atribuição de valores verdade (verdadeiro ou falso) para um conjunto de variáveis booleanas que torne uma fórmula lógica verdadeira. Embora seja extremamente eficiente para problemas puramente lógicos usando operadores como AND, OR e NOT, o SAT possui baixa expressividade para lidar diretamente com construções complexas de linguagens de programação, como aritmética de inteiros ou pontos flutuantes.

Para superar essa limitação, utiliza-se o \textbf{Satisfatibilidade Módulo Teorias (SMT)}, que é uma extensão do SAT. O SMT integra o poder de decisão booleana do SAT com teorias matemáticas específicas (como vetores de bits, arrays e números reais). Isso permite que o \textit{solver} compreenda e verifique propriedades sobre tipos de dados da linguagem, como \texttt{int} e \texttt{float}, e operações aritméticas. No contexto do ESBMC, o código C ou Python é traduzido em fórmulas SMT que são verificadas por \textit{solvers} modernos como Z3 ou Bitwuzla.

\section{O Verificador ESBMC}

O ESBMC (\textit{Efficient SMT-Based Context-Bounded Model Checker}) é um verificador de código aberto que estende o BMC para programas multicore e sistemas dinâmicos. Ele suporta linguagens como C, C++, Python e CUDA. 

O fluxo de trabalho do ESBMC consiste em:
\begin{itemize}
    \item \textbf{Frontend}: Análise léxica e sintática do código-fonte;
    \item \textbf{GOTO-Program}: Transformação em um grafo de fluxo de controle simplificado;
    \item \textbf{Execução Simbólica}: Geração de equações que representam a semântica do programa;
    \item \textbf{SMT Encoding}: Codificação das equações em fórmulas para \textit{solvers} como Z3, Bitwuzla ou Boolector.
\end{itemize}

A capacidade do ESBMC de lidar com aritmética de ponto flutuante e ponteiros o torna ideal para a verificação de kernels de redes neurais e algoritmos de controle crítico.

\section{Primitivas de Modelagem C/C++ no ESBMC}

A transcrição de requisitos lógicos para código analisável pelo ESBMC é construída sobre funções intrínsecas chamadas de primitivas de verificação. Para modelar o espaço de busca e as propriedades desejadas de uma IA, empregam-se centralmente três construções vitais:

\begin{itemize}
    \item \textbf{Valores Não-Determinísticos (\texttt{nondet})}
    
    Permitem ao \textit{solver} instanciar qualquer valor possível para o tipo da variável, forçando a ferramenta a testar matematicamente todo o domínio computacional.
    \begin{lstlisting}[language=C, numbers=none]
float input_val = nondet_float();
    \end{lstlisting}

    \item \textbf{Restrições de Hipótese (\texttt{\_\_ESBMC\_assume})}
    
    Restringe o espaço não-determinístico às pré-condições do problema. O comando orienta o verificador a ignorar caminhos simbólicos que não atendam à condição. Em notação formal, modela uma fronteira que força as entradas $x$ a estarem contidas num conjunto seguro $\mathcal{S}$ do domínio: $x \in [-10, 10]$.
    \begin{lstlisting}[language=C, numbers=none]
__ESBMC_assume(input_val >= -10.0f && input_val <= 10.0f);
    \end{lstlisting}

    \item \textbf{Asserções de Propriedade (\texttt{\_\_ESBMC\_assert})}
    
    Expressa as invariantes e \textit{post-conditions} de segurança. Se o ESBMC encontrar iterativamente qualquer valor de $x$ (confinado pelo \texttt{assume}) que viole o \texttt{assert}, a prova matemática "quebra", reportando o caminho indutivo como \textit{Counter-Example Trace}. Matematicamente, exige que a função $f(x) = y$ nunca escape do conjunto limite seguro $y_{max}$.
    \begin{lstlisting}[language=C, numbers=none]
float y = model_predict(input_val);
__ESBMC_assert(y < 10.0f, "Saida excedeu o limite teorico");
    \end{lstlisting}
\end{itemize}

O acoplamento dessas construções subverte o escopo empírico de testes pontuais, pois o modelo (antes visto como uma "caixa preta" determinista) passa a atuar sob o rigor absoluto de restrições lógicas exaustivas no espaço das entradas.

\section{Redes Neurais Quantizadas (QNN)}

Uma Rede Neural Artificial tradicional treina seus pesos sinápticos e viéses (\textit{biases}) utilizando representação numérica de Ponto Flutuante (geralmente 32 ou 64 bits, previstos no IEEE 754). Isso entrega capacidade de simular distribuições muito suaves (curvas complexas), mas consome enorme potência de processamento e RAM.

Uma QNN (\textit{Quantized Neural Network}) converte essa rede para Ponto Fixo com tamanho estrito de palavras de hardware (Ex: 8-bits ou 16-bits inteiros). 
O funcionamento consiste em:
\begin{enumerate}
    \item \textbf{Escalonamento Geométrico:} O valor float real (ex: $0.5$) é multiplicado por um fator de escala (ex: $256$ para 8-bits de fração) para se tornar num número Inteiro rígido ($128$).
    \item \textbf{Aritmética Truncada:} Todas as somas e multiplicações entre os neurônios passam a utilizar registradores inteiros normais, o que é brutalmente mais rápido e econômico energeticamente na ponta (\textit{Edge AI} / IoT).
    \item \textbf{Realinhamento:} Sempre que dois números de ponto fixo são multiplicados (o que eleva a escala ao quadrado), os \textit{bits} menos significativos são descartados (\textit{Shift-Right} lógico \texttt{>>}) para alinhar a palavra de volta à escala padronizada.
\end{enumerate}

O truncamento geométrico supracitado invariavelmente perde a precisão infinitesimal natural da IA. Ferramentas de verificação formal, como o ESBMC, são empregadas para analisar se a rede neural sofre degradação dessa quantização ao ponto de desclassificar uma predição crítica ou se sofrerá um \textit{overflow} de memória sob ataque adversário.
